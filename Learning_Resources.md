# 推奨学習リソース：初心者向け SRE 独学ロードマップ (2026年版)

「Quiet Growth（静かな成長）」をテーマに、最小限の投資で現場で通用するスキルを身につけるための厳選リソースです。

---

## 🛠 Phase 1: 土台を固める（Linux & Network）
派手なツールの前に、OSと通信の仕組みを「体感」レベルで理解します。

### 📚 書籍
- **『[試して理解] Linuxのしくみ 第2版』** (武内 覚 著)
  - **理由:** インフラエンジニア・SRE必読の書。OSの挙動を実験を通して学べる。
- **『1冊ですべて身につくネットワーク入門・構築』** (技術評論社)
  - **理由:** 図解が豊富で、基礎から実務的な構築までカバー。

### 🎥 動画・Web
- **[Udemy] Linuxサーバー構築入門**
  - **理由:** コマンドライン操作に慣れるための実践的なコース。
- **Ubuntu Server 自宅ラボ構築**
  - **方法:** 古いPCやミニPCにインストールし、SSHで操作。

---

## 🚀 Phase 2: コンテナと自動化（Docker & Terraform）
「手作業」を「コード」に置き換える感覚を養います。

### 📚 書籍
- **『仕組みと使い方がわかる Docker&Kubernetesのきほんのきほん』** (小笠原 種高 著)
  - **理由:** 概念理解の第一歩として最適。
- **『Terraformの教科書』** (技術評論社)
  - **理由:** IaC（Infrastructure as Code）の基礎と実践を体系的に学べる。

### 📝 記事・実践
- **[Zenn] Terraform入門** (shonansurvivors 著)
  - **実践:** 自分のポートフォリオサイトを Docker 化し、Terraform で AWS/Google Cloud のリソースを定義。

---

## 🏗 Phase 3: SREの本質（Reliability Engineering）
「壊れない」ではなく「どう壊れ、どう直すか」を管理します。

### 📚 書籍
- **『SRE サイトリライアビリティエンジニアリング』** (オライリー・ジャパン)
  - **理由:** 通称「Google SRE本」。SREの哲学と原則を学ぶためのバイブル。
- **『入門 監視』** (Mike Julian 著)
  - **理由:** 「何を監視すべきか」という設計思想を学べる。
- **『Kubernetes完全ガイド 第2版』** (青山 真也 著)
  - **理由:** 日本語で最も詳細なk8s解説書。

### 🎥 動画・実践
- **[YouTube] Cloud Girl (AWS解説アニメ)**
  - **理由:** 複雑なクラウドの概念を直感的に理解できる。
- **GitHub Actions による CI/CD 構築**
  - **理由:** デプロイの自動化を体験する。
- **Prometheus/Grafana による可視化**
  - **理由:** SLO/SLI を設定し、メトリクスを監視する。

## 💡 2026年の学習アドバイス
1. **AIの活用:** YAMLなどの定型コード作成は AI に任せ、**「デバッグ能力」と「アーキテクチャ設計力」**を磨く。
2. **アウトプット:** 学んだことを **Zenn** や **GitHub** に淡々と記録し、ポートフォリオ化する。
